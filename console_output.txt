Issue 1: getFacebookPixelId() is likely returning undefined on Goli.com

Your pixel ID resolution order is:

data-meta-pixel-id on #rcart-widget-root
data-facebook-pixel-id on #rcart-widget-root
VITE_FACEBOOK_PIXEL_ID build-time env var
On your dev store, one of these is populated. On Goli.com, if #rcart-widget-root doesn't have the data-meta-pixel-id attribute set correctly, and the build-time env var is baked in for a different store, getFacebookPixelId() returns undefined — and both callFbq and pageview silently no-op.

Quick check: In the console on Goli.com run:

document.getElementById('rcart-widget-root')?.dataset
Verify metaPixelId or facebookPixelId is present and matches 274758730140253.

Issue 2: window.Shopify?.analytics?.publish is not the Web Pixel API

This is the critical one. In your pageview and trackWithConfiguredPixel functions you're calling:

window.Shopify?.analytics?.publish?.("rcart_pageview", {...})
But window.Shopify.analytics.publish is the storefront Customer Events API — it's available in the browser context. Your web pixel extension uses analytics.subscribe from the sandboxed Web Pixel API, which is a completely separate context (isolated iframe). These two analytics objects are different — Shopify bridges them internally.

The problem is: window.Shopify.analytics may not be available or may not be initialized yet when your pageview() call runs. If it's undefined at call time, the publish silently fails and your web pixel extension never receives the event, so browser.sendBeacon never fires.

Check in console on Goli.com:

window.Shopify?.analytics
// Should return an object with a publish method
// If undefined — that's your bug
If it's undefined, it means your pageview() is being called before Shopify's Customer Events bridge is ready.

The fix: defer publish until Shopify analytics is ready

function publishToShopify(eventName: string, data: Record<string, unknown>) {
  if (window.Shopify?.analytics?.publish) {
    window.Shopify.analytics.publish(eventName, data);
    return;
  }
  // Not ready yet — retry after a short delay
  setTimeout(() => {
    window.Shopify?.analytics?.publish?.(eventName, data);
  }, 500);
}
Or better, listen for the DOM to be fully ready before firing pageview:

export const pageview = (queryParams: Record<string, unknown> = {}, eventId?: string) => {
  const id = eventId || generateEventId();
  if (typeof window === "undefined") return id;

  ensureFbqInitialized();
  const pixelId = getFacebookPixelId();
  if (typeof window.fbq === "function" && pixelId) {
    window.fbq("trackSingle", pixelId, "PageView", queryParams, { eventID: id });
  }

  // Defer Shopify publish to ensure the bridge is initialized
  const doPublish = () => {
    window.Shopify?.analytics?.publish?.("rcart_pageview", { queryParams, eventId: id });
  };

  if (document.readyState === "complete") {
    doPublish();
  } else {
    window.addEventListener("load", doPublish, { once: true });
  }

  return id;
};
